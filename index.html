Category Two: Algorithms and Data Structures

The algorithms and data structures for this application seem to be very simple. An algorithm is used to display the items from the inventory if the user’s credentials are matched. That is after the user is logged in, all the items from the inventory are displayed then, he or she is prompted to three different options from the available mechanisms; add or remove items, increase or decrease items, then setting up alert when a specific item reaches zero. I plan on enhancing the algorithm to better retrieve information from the inventory by modifying the data structures using array lists to store the inventory items. Category Three: Databases

The inventory management application is simple app that was built to manage inventory items in warehouse. The main users for the application are people who work in warehouse/store which will be in control of inventory, such as store managers, and any authorized employees. The application should be able to hold as many users as possible and users will be able to view, add and remove items from the store.
The application has a security feature which means only users who have an account will have access to the content of the application. If users do not have an account, they will be prompt to create their account with their credentials, otherwise the application will not allow them to access any of its content. The application uses text file to store the users’ information and the files will be stored in a local storage on the local machine, which forces the user to only access the application with the specific device. My goal is to have those text files stored on a web-based database to provide more flexibility for the user, that is the user can access the application if downloaded across other devices. Also, having this flexibility will make it easier for the backend to do any improvements to the application when necessary. However, I am concerned about some artifacts that I think need some improvements as I am not very experienced on building software applications. I know that my application needs to have a more structured coding practice, better storage methods and data structures. I plan on making these improvements to the application to assure a better result after launching.

Based on some research I did on the method I use to store the users’ information; I found some downsides to that method even though it seems to have good advantages. My first concern is since these files will be stored locally, this makes the application more vulnerable and unauthorized people can easily hack some sensitive data such as, usernames and passwords. Also, managing these types of data does not seem to be an easy task after having several users. Whereas, web-based databases today offer several features that can handle these tasks better and more secure.

This was not an easy step within the enhancements that had to do. I had different ideas on which database that I was going to use for the application, then I came across an easy, user friendly web-based database called XAMPP. I was excited as I started progressing through this improvement. I went ahead and created several examples to see what the data would look like by creating tables to store the information. For the users, I had a user table with 3 columns to hold their name, password, employee ID. Then, I created other tables to test out the items by having an item table with category, quantity, price, and date. Therefore, these enhancements will make the inventory application more secure and easy to manage and organize the data being stored. One main advantage of the web-based database, it will be simple to import and export files, backups can also be done to assure any loss of data. At this point, most of goals for a database have been met and I will keep learning more on how to organize the data and make it look clean, easy to read and manage.

The purpose of the inventory application is, to track inventory in a warehouse. The targeted users are anyone who works at a warehouse and is in control of inventory, such as store manager, stocker, and other authorized employees. The application has the following requirements:
 A screen with user login, with the option of creating an account for first time users.
 A grid that shows the inventory items in the store with the mechanism to add and remove items in different categories.
 A screen that shows the items to add to grid.
 A grid with the database.
 Also, a mechanism is required to send an alert to the user when a specific item has reached zero.
For this inventory application, I developed it from scratch and as I am going through it after getting hint on code review, I find some areas that needed to be enhanced, to be added and/or other parts of code to be removed in order to enhance its working. Learning software design specifically in this course, helps me better understand the design process, database and data structure I realize that I can make the application more interesting and attractive to users. I started by commenting the code as it is vital in computer programming, this project has really helped improving my skills a bit higher since almost all the files needed to be commented fully, that way if anyone who does not know anything about coding can better understand the function of each block of code in the whole project. Also, since logical errors cannot be detected by the compiler this makes it a little harder to remove logic errors and may even leave the deployed application with the errors. Additionally, the project has given me the opportunity to work with external files. In the previous version of the application I did not properly include external files, like text files or database files where history will be stored for future reference. The inventory application will be more reliable since it will keep the items of each product in a specific category and they will be displayed in a grid whenever the user wants to. I think with the MySQL database I can keep the product descriptions, dates added and other important information necessary for inventory. To finish, the previous application did not include a background and it didn’t seem to be attractive to the users. While going through each block of the code, I found it more interesting to add a background and have a different color scheme. These enhancements in the program has helped improving my skills in software design, data structure and database since after the code review process I then had the opportunity to go through the project and with the help of the code review checklist I identified several mistakes, I then corrected them and hope they don’t happen in the future.
